## Four Strategic Pillars

1. **Cloud-Native** (C, J) - Kubernetes-first, microservices architecture
2. **Observability** (O) - OpenTelemetry, distributed tracing, metrics
3. **NLP/Natural Language** (N, U) - Semantic search, language understanding
4. **Efficiency** (G, T, E) - SIMD acceleration, tunable performance, granular control

---

## Elevator Pitch

> **CONJUGATE** is a cloud-native distributed search engine that combines observability, natural language understanding, and granular analytics in a highly tunable architecture. It provides 100% OpenSearch API compatibility while delivering 4-8× faster performance through SIMD acceleration and advanced compression.

## Conceptual Meaning

### Mathematical Conjugates
In mathematics, **conjugate pairs** (e.g., `a + bi` and `a - bi`) work together to achieve balance and produce real-valued results.

### Application to CONJUGATE
Just as mathematical conjugates work in harmony, **CONJUGATE components work together seamlessly**:
- Cloud + Observability = Self-monitoring infrastructure
- Natural Language + Understanding = Semantic comprehension
- Granular + Tunable = Precision with flexibility
- Analytics + Engine = Insights powered by performance

The name represents **harmony, balance, and coordination** across distributed systems.

---

## Why CONJUGATE?

### Migration from "Quidditch"
The project was rebranded from "Quidditch" to "CONJUGATE" in January 2026 due to:
- **Trademark concerns**: "Quidditch" is a registered trademark of Warner Bros
- **Professional positioning**: Technical name better communicates capabilities
- **Strategic clarity**: Emphasizes four key pillars (Cloud, Observability, NLP, Efficiency)

See **[NAMING.md](NAMING.md)** for complete naming rationale and **[MIGRATION.md](MIGRATION.md)** for migration guide.
